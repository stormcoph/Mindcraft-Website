About Mindcraft

Mindcraft creates intelligent AI agents for Minecraft using Large Language Models and Mineflayer. These bots can understand natural language, learn from their environment, and execute complex tasks in the game world.

Whether you're looking to experiment with AI, create helpful assistants, or just add some company to your Minecraft world, Mindcraft makes it easy to bring intelligent bots into your game.

Get Started

Requirements

  • Minecraft Java Edition (up to v1.21.1, recommend v1.20.4)
  • Node.js Installed (at least v14)
  • One API key from: OpenAI, Gemini, Anthropic, Replicate, Hugging Face, Groq, or Ollama installed

Installation Steps

  1. 1

    Clone the Repository

    git clone https://github.com/kolbytn/mindcraft.git
  2. 2

    Configure API Keys

    Rename keys.example.json to keys.json and add your API key

  3. 3

    Install Dependencies

    npm install
  4. 4

    Start Minecraft

    Start a Minecraft world and open to LAN on port 55916

  5. 5

    Run Mindcraft

    node main.js

Documentation

Bot Profiles

Bot profiles are JSON files (such as andy.json) that define:

  • Bot backend LLMs to use for talking, coding, and embedding
  • Prompts used to influence the bot's behavior
  • Examples help the bot perform tasks
{
  "name": "Andy",
  "model": "gpt-4o",
  "code_model": {
    "api": "openai",
    "model": "gpt-4"
  },
  "embedding": {
    "api": "openai",
    "model": "text-embedding-ada-002"
  }
}

Server Connection

Local Server

By default, Mindcraft connects to a local Minecraft server on port 55916.

Online Servers

To connect to online servers, your bot needs a Microsoft/Minecraft account.

// In settings.js
"host": "111.222.333.444",
"port": 55920,
"auth": "microsoft"

⚠️ Important: The bot's name in profile.json must exactly match the Minecraft profile name!

Developer Reference

Docker Container

For safer code execution, run in a Docker container:

docker run -i -t --rm -v $(pwd):/app -w /app -p 3000-3003:3000-3003 node:latest node main.js

# Or simply
docker-compose up

Command Line Profiles

Specify profiles via command line:

node main.js --profiles ./profiles/andy.json ./profiles/jill.json

Patches

To add a patch, change your local node module file and run:

npx patch-package [package-name]

Frequently Asked Questions

Is it safe to use code execution?

Code execution is disabled by default for safety. If enabled, the code is sandboxed but still vulnerable to injection attacks. We recommend using Docker when enabling code execution and never connecting to public servers with code execution enabled.

What Minecraft versions are supported?

Mindcraft supports Minecraft Java Edition up to v1.21.1, with v1.20.4 being the recommended version for best compatibility.

Which LLM provider should I use?

Different LLMs have different strengths. OpenAI's models tend to perform well for general tasks, Claude excels at following complex instructions, and local models like those from Ollama offer privacy and no usage costs. We recommend trying different models to see which works best for your use case.

Can I connect to online multiplayer servers?

Yes, but you'll need an official Microsoft/Minecraft account for your bot. Configure the host, port, and authentication in settings.js. Remember that the bot's name in the profile.json must exactly match the Minecraft profile name.

How do I customize my bot's personality?

Edit the bot's profile JSON file to customize prompts, examples, and behavior instructions. You can create entirely new profiles or modify existing ones like andy.json.
